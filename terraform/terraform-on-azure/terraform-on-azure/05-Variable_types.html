In this exercise, you will learn the different types of terraform variables. Variables are used in terraform in order to avoid hard coding values into our configuration. There are multiple types of these variables, so we will explore them all in this lab. #Outcomes You should be able to: - Explore terraform variables - Use some of them #Steps 1. Instead of using hard coded values into our configuration files, terraform provides us with a declarative method for declaring our Variables. Without variables, we would have files like this ``` vi main.tf ``` ``` resource "local_file" "jedi" { filename = "/tmp/first-jedi" content = "phanius" } ``` Instead, we can use a main.tf that look more like This ``` resource "local_file" "jedi" { filename = var.jedi["filename"] content = var.jedi["content"] } ``` and then declare our variables in a seperate file called `variables.tf`. This file is used generally for declaring variables only, but we can fill the values in there too: ``` vi variables.tf ``` ``` variable "jedi" { type = map default = { filename = "/tmp/first-jedi" content = "phanius" } } ``` Now, let's run `init` to install the local provider. ``` terraform init ``` ``` Initializing the backend... There are also `bool`, `string`, `number`, `lists`, `sets` and `objects`. `String variable`: ``` variable "name" { type = string default = "Mark" } ``` In order to fetch the value of the name variable later on in the main.tf file, we can use this: ``` var.name ``` `Bool variable`: ``` variable "vaccinated" { type = bool default = true } ``` `Number variable`: ``` variable "age" { type = number default = 35 } ``` `Map variable`: ``` variable "hard_drive" { type = map default = { slow = "HHD" fast = "SSD" } } ``` In order to fetch the value of the slow key in hard_drive variable later on in the main.tf file ``` var.hard_drive["slow"] ``` `List variable`: ``` variable "gender" { type = list(string) default = ["Male", "Female","Non-binary"] } ``` In order to fetch the value of the second element in gender list of strings later on in the main.tf file ``` var.gender[1] ``` `Set variable`: ``` variable "users" { type = set(string) default = ["tom", "jerry", "pluto", "daffy", "donald", "jerry", "chip", "dale"] } ``` The difference between set and list is that a set can't have duplicated elements. Let's correct the mistake in the previous set by deleting one occurance of "jerry" ``` variable "users" { type = set(string) default = ["tom", "jerry", "pluto", "daffy", "donald", "chip", "dale"] } ``` We will explore the relationship between main.tf files, and the variable files in the next lab! # Terraform on Azure - Input Variables Basics - Create / Review the terraform manifests - c1-versions.tf - c2-variables.tf - c3-resource-group.tf - c4-virtual-network.tf 1.
